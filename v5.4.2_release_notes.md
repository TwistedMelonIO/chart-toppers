# v5.4.2 Release Notes — Unified Auto-Test Entry Point

**Released: April 29, 2026**

## What's New

### `scripts/test-all.sh` — single command to run every auto-test
- **Summary**: New wrapper script runs the full automated test suite in order: license persistence first, game-logic round flow second.
- **Details**:
  - Suite 1: `test-license-persistence.sh` (12 checks, ~30s)
  - Suite 2: `test-rounds.py` (10 game-flow scenarios)
  - License tests run first and fail-fast — if persistence is broken, the destructive game tests are skipped to avoid leaving the container in an unknown state.
- **Usage**: `bash scripts/test-all.sh` — one command, full coverage.

### `TEST_CHECKLIST.md` — preamble updated
- Now points at `test-all.sh` as the canonical entry point.
