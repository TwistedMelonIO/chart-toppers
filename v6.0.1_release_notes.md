# v6.0.1 Release Notes — Selftest checklist + repo housekeeping

## Summary

Folds in pending product work that was sitting unstaged when v6.0.0 shipped, plus repo housekeeping so generated PDFs and pack-data backups stop showing up as untracked noise.

## Changes

- **`/api/selftest/checklist`** rewritten as a **show-flow checklist** (Pre-show → Opening → Round 1 → … → Outro) instead of the previous developer-style round/QLab item list. Makes the in-app test report read the way the show actually runs.
- **`DEEP_TEST.md`** — full deep-test plan committed to the repo so it isn't only on the dev machine.
- **`PRIORITY_TESTS.md`** — top-5 priority test checklist for QLab dry runs.
- **`scripts/test-rounds.py`** — automated round-flow smoke test that drives the running server via the HTTP API.
- **`.gitignore`** — ignores `reports/*.pdf`, `test-report-demo.pdf`, `Chart_Toppers_*.pdf`, and `data/packs/_backup_*` / `_pre_*`.
- Removed the stale `reports/Pack1_TrackReport_2026-05-01.pdf` that had been committed.

```bash
docker compose up -d --build
```

License keys, machine IDs, pack settings, and activity log persist across the rebuild.
