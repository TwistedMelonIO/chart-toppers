# v5.4.3 Release Notes — License Persistence in Test Report

**Released: April 29, 2026**

## What's New

### License Persistence checks added to `/api/selftest`
- **Summary**: The runtime self-test endpoint (consumed by Docforge) now reports on the dual-tier license persistence introduced in v5.4.0.
- **New checks** (3, all under category "License Persistence"):
  - **Volume copy present** — `/app/data/license_key` exists
  - **Host backup present** — `/app/host-data/license_key` exists
  - **Tiers in sync** — both files contain identical content
- **Impact**: When generating an installation test report PDF in Docforge, persistence health appears alongside the existing infrastructure / scoring / cue config sections. Total automated checks: 17 → 20.

## Technical Changes
- `src/server.js`: 3 new `addResult` calls inside `/api/selftest` using existing `_safeRead()` helper.
