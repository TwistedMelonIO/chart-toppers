# v5.8.0 Release Notes — R4 iPad single-answer mode

## Summary

Round 4's iPad stage host view now shows a single answer at a time, driven by the `/chart-toppers/r4/{n}` OSC. The iPad stays blank on R4 entry and updates to show the band/track for whichever R4 slot QLab is playing.

## Changes

- **`src/server.js`** — `resetR4Tracks()` no longer auto-emits `currentTrack` for slot 1 on R4 entry. R4 plays in arbitrary order, so the iPad waits for OSC.
- **`src/server.js`** — `playR4Track()` now emits `currentTrack` with the played slot number, so each `/chart-toppers/r4/{n}` updates the iPad to that single answer.
- **`public/stagehost.js`** — `answersUpdate` handler trusts the payload's `round` field instead of the client's stale `currentRound`. Fixes a race where R4 entry briefly rendered the full 35-track list (because `answersUpdate` from `resetR4Tracks` arrives before `roundUpdate`).

## Behavior

| Event | iPad |
|-------|------|
| Enter Round 4 | Blank |
| `/chart-toppers/r4/1` arrives | Shows R4T1 answer ("Answer 1 of 35") |
| `/chart-toppers/r4/7` arrives | Shows R4T7 answer ("Answer 7 of 35") |
| Incorrect/pass during R4 | Existing index-advance behavior unchanged |

**Version**: 5.8.0
