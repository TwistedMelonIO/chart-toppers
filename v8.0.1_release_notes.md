# Chart Toppers v8.0.1

## R4 Tiebreaker — buzzer pause / incorrect resume / correct stop

The Round 4 tiebreaker (cue `TB`) now responds to buzzer flow:

- **Buzz** → pauses the TB cue (was: nothing).
- **Incorrect** → resumes the TB cue from the paused position (was: nothing).
- **Correct** → stops the TB cue (unchanged).

Operator can now run repeated buzz / incorrect cycles during a tiebreaker — the track pauses each time a team buzzes, resumes if they get it wrong, and only stops when a team finally gets it right.

No changes to R1, R2, R3, or to the rest of R4 normal play. Buzzer arming during tiebreak (already handled by `tiebreakActive` skipping the defensive disarm) is unchanged.

## Files changed
- `src/server.js` — buzz handler + incorrect handler, ~6 lines added.
- `package.json`, `public/index.html`, `public/settings.html` — version bump.
