# v7.3.1 Release Notes — Auto-seed audioBasePath from env

**Released: May 2, 2026**

## What's Fixed
- **Fresh installs now pick up the audio path automatically.** When `pack-settings.json` has no `audioBasePath` (new machine, first boot), the server now falls back to the `QLAB_AUDIO_PATH` env var that `install_license.sh` already writes to `.env`. No more "type the same path into Settings after install" step.

## Technical Changes
- `src/server.js` `loadPackSettings()`: after loading (or if no settings file exists), seeds `packSettings.audioBasePath` from `process.env.QLAB_AUDIO_PATH` when still empty. Logged as `[PACK] Seeded audioBasePath from QLAB_AUDIO_PATH env: "..."`.
