# v5.5.0 Release Notes — R4 Disarm, OSC Reliability, Track QA

**Released: April 30, 2026**

## What's New

### Round 4 — Auto-disarm last track for second team
- **Summary**: When the second team starts in R4, the server disarms the QLab cue matching the last track the first team played.
- **Details**: Server tracks the most recent successful R4 track-play OSC. On `/chart-toppers/playing/{team}` for the second team, it sends `/cue/R4TG{N}/armed 0` to QLab.
- **Impact**: Player 2's playthrough automatically skips Player 1's last track without operator intervention.

### Track QA tool (`/qa`)
- **Summary**: Hidden batch-mode audio verification tool for testing every track across every pack through the real show cues.
- **Details**: Pick a pack and round, server loads sequential batches into the show's actual `R1T*`, `R2T*`, `R3T*`, `R4T*` cues (paths, names, answer notes). Confirm each batch to advance; persisted across container restarts.
- **Impact**: Operator can audit hundreds of tracks systematically when packs are added or updated, without polluting the operator dashboard.

### Audio path quote-stripping
- **Summary**: The audio base path setting now sanitises wrapping single/double quotes on save and on load.
- **Impact**: Pasting a path from Finder/Terminal (which often quotes paths containing spaces) no longer breaks QLab cue file targets.

### R4 re-shuffle on pack change
- **Summary**: Switching packs now re-shuffles Round 4 with the new pack's tracks.
- **Impact**: R4 cues no longer sit in pack order after a pack swap; iPad answers and QLab cues stay consistent.

## Bug Fixes

- **OSC stagger across all bulk operations**: `resetR1Tracks`, `resetR4Tracks`, `loadGenreTracks` (R1/R2/R3 branches), `resetRounds`, and `resetAll` now stagger every OSC message by `OSC_STAGGER_MS` (20 ms). QLab no longer drops messages when ~100+ commands previously fired back-to-back, fixing stale cue notes after shuffles and resets.
- **Cancellable shuffle queues**: Each staggered burst tracks its `setTimeout` handles. A subsequent reset/load cancels the previous queue's tail-end writes so they can't clobber the new batch.
- **R1 auto-shuffle removed on round entry**: The genre picker is now the single source of R1 randomness — auto-shuffle on entry was racing with genre-pick OSC writes and producing inconsistent QLab state.
- **R4TF cue notes now reliably match track names** after shuffles.

## Technical Changes
- New helpers `scheduleOsc` / `clearTimeoutBucket` for cancellable OSC pacing.
- New persistence file `data/track-qa.json` (gitignored).
- New routes: `/api/qa/state`, `/api/qa/select`, `/api/qa/confirm`, `/api/qa/back`, `/api/qa/reload`, `/api/qa/reset-round`.
- Frontend: `public/qa.html`, `public/qa.js`.
