# v6.1.0 Release Notes — Buzzer ships as a real .app

## Summary

The buzzer is now packaged as a proper macOS application — `~/Applications/QLab Buzzer.app`. Granting Accessibility permission is now **one click per Mac, forever**: just toggle on "QLab Buzzer" in System Settings, no more hunting through `/opt/homebrew/Cellar/...` paths.

## What changed

- **`buzzer/build_app.sh`** — new script. Builds `QLab Buzzer.app` via PyInstaller, sets `LSUIElement=true` (no Dock icon), bundles `config.json`, and ad-hoc signs.
- **`buzzer/install_service.sh`** — now builds the .app, installs it to `~/Applications/QLab Buzzer.app`, and registers the LaunchAgent to launch the .app's binary directly (so launchd can `KeepAlive` it).
- **`buzzer/qlab_buzzer.py`** — uses `sys._MEIPASS` when frozen so the bundled `config.json` is found at runtime.
- **`.gitignore`** — excludes `buzzer/build/`, `buzzer/dist/`, and the generated `QLab Buzzer.spec`.

## On every show machine (one-time)

After `git pull` + `./buzzer/install_service.sh`, the installer prints:

```
System Settings → Privacy & Security → Accessibility
  1. Click the +
  2. Add: ~/Applications/QLab Buzzer.app
  3. Toggle it ON
```

That's it. No symlink chasing, no Cellar path hunting. The .app survives Python upgrades, repo moves, and reboots.
