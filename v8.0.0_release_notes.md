# v8.0.0 Release Notes — Production-locked cut

**Released: May 2, 2026**

Major version bump to mark this as the locked-in production build after a full show run-through and multi-machine deploy validation. No code changes from v7.3.11 — this release consolidates the 7.3.x stabilisation series into a major version.

## Highlights of the 7.3.x series rolled into 8.0.0

### New features
- **R1 explicit iPad track OSC** (v7.3.8): `/chart-toppers/r1t1` … `/r1t6` set the stage host iPad to a specific track. Auto-advance disabled for R1 (v7.3.9) — iPad only moves on these explicit OSCs.
- **Buzzer permission canary on dashboard footer** (v7.3.0): amber pulse + "No input — check Mac permissions" if the buzzer daemon connects but receives zero keypresses for 30s.
- **R3 SD always returns to page 12** (v7.3.6) on correct or incorrect answer.
- **R4 disarms ALL first-team tracks** (v7.3.7) for the second team's pass, not just the most recent.

### Reliability fixes
- **OSC bridge handles no-arg messages** (v7.3.2): `/cue/GBUZZER/start` and other no-arg cue triggers now actually fire (bridge was returning HTTP 400).
- **Auto-seed audioBasePath from `QLAB_AUDIO_PATH` env** (v7.3.1): fresh installs work without re-typing the path in Settings.
- **`/refreshtracks` always navigates SD on team swap** (v7.3.10): removed an over-zealous suppress flag that broke the operator's refresh flow.
- **R3 mashup iPad list capped at 56px** (v7.3.11) — 80s/2000s now match 1990s sizing.

### Setup tooling
- **`scripts/setup-buzzer-host.sh`**: one-shot per-machine setup with `tccutil reset` for stale Python TCC entries.
- **Bulk buzzer SCP/LaunchAgent deploy** (v6.2.2 baseline, still in place).

## Operator notes for new-machine deploys
1. Run `setup-buzzer-host.sh` for buzzer + permissions.
2. `install_license.sh` will prompt for and write `QLAB_AUDIO_PATH` to `.env`.
3. Copy the **full** `audio/` folder to the host path the workspace expects (QLab's bundle export drops untargeted files — see memory `feedback_qlab_bundle_drops_untargeted_audio.md`).
4. Verify R4 group cues in QLab are numbered `R4TG1` … `R4TG35` so the second-team disarm lands.
