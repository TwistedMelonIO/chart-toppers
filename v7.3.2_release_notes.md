# v7.3.2 Release Notes — OSC bridge handles no-arg messages

**Released: May 2, 2026**

## What's Fixed
- **Golden Record sting (and any other no-arg cue trigger) now actually fires.** The OSC bridge previously rejected `JSON.stringify({address, value: null})` with HTTP 400 because `buildOscMessage` only handled number/string/array values. The chart-toppers server uses `null` for no-arg messages like `/cue/GBUZZER/start`, so those silently never reached QLab. Bridge now treats `null`/`undefined` as a valid no-arg OSC message (`,` typetag).

## Technical Changes
- `bridge/server.js` `buildOscMessage()`: short-circuits `value === null || undefined` to a no-arg OSC packet (`address` + `,` typetag, no payload).
