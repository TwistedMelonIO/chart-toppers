# Chart Toppers v8.0.4

## Fully offline-safe frontend assets

The main scoring dashboard hung on machines with no internet because `index.html` loaded the Lucide icon library from `unpkg.com` as a render-blocking script, and all three pages pulled fonts from `fonts.googleapis.com`. With no upstream internet, the browser stalled waiting for those external requests to time out.

The stage-host page was unaffected (only fonts, which the browser handles non-blocking enough to render) — so the symptom was "stage host loads, main UI hangs" on offline routers.

## Fix

All external CDN references removed. Assets are now bundled with the image:

- `public/vendor/lucide.min.js` — Lucide icons (~400 KB)
- `public/vendor/fonts.css` + `public/vendor/fonts/*.woff2` — Bebas Neue, Chakra Petch (500/600/700), Outfit (300/400/500/600/700), Latin subset only

`index.html`, `settings.html`, and `stagehost.html` all reference the local copies. The app now works fully offline from a fresh container with no cache.

## Files changed
- `public/vendor/` — new directory with lucide + fonts
- `public/index.html`, `public/stagehost.html`, `public/settings.html` — replaced CDN links with local paths
- `package.json`, `public/index.html`, `public/settings.html` — version bump to 8.0.4
