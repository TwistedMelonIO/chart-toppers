# Chart Toppers v8.0.2

## R4 final-decision now compares R4 points only

The end-of-R4 winner / tiebreaker decision (DUALGO retarget + `/chart-toppers/dualscreen` handler) previously compared **cumulative score** = `earnedTime + points` across all four rounds. That meant a Round 4 ending 6-6 in points still routed to the winner-reveal cue if the cumulative R1-R3 time differed between teams.

This release adds a dedicated `r4FinalRanking()` helper that compares **R4 points only** for the two paths that actually fire the DUALGO cue:
- `retargetDUALGO()` — chooses target between `TIEBREAK` and `R5`.
- `/chart-toppers/dualscreen` handler — chooses to set `tiebreakActive` + arm buzzers, or fire winner reveal.

If R4 ends with both teams on the same number of points (e.g. 6-6), the tiebreaker fires regardless of how the time tally landed in earlier rounds. If R4 points differ (e.g. 7-6), DUALGO targets R5 (winner reveal).

`teamRanking()` is unchanged — it still uses cumulative score for everything else (R2/R3 coin-flip detection, mid-game LEADERSD updates, R4 first-team rule, etc.).

Verified on machine 216 (MSC Poesia) before tagging.
