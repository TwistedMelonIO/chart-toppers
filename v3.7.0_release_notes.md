# v3.7.0 Release Notes — QLab Enhancements

**Released: April 3, 2026**

## What's New

### Round 1 Cue Labels
- **Summary**: Short and long clips now labelled in QLab for easy identification
- **Details**: Round 1 hook cues show `[SHORT]` and reveal cues show `[LONG]` appended to the cue name
- **Impact**: Operators can quickly identify and re-trigger the correct clip during gameplay

### Round 1 Answer Notes
- **Summary**: Correct answers embedded in QLab cue notes for short clips
- **Details**: Each Round 1 short clip (`.1`) cue now has `Correct Answer: Artist - Track` in its QLab notes field
- **Impact**: Operators can see the answer at a glance without needing to cross-reference

### Reset All Arms QLab Cues
- **Summary**: Full game reset now arms all cues in the QLab workspace
- **Details**: When Reset All is triggered (dashboard, API, or OSC), an OSC command `/cue/*/armed 1` is sent to QLab
- **Impact**: No need to manually re-arm cues between games

## Technical Changes
- Added `updateQLabCueNotes()` function for setting QLab cue notes via OSC
- Added `armAllQLabCues()` function called during `resetAll()`
- Round 1 cue naming updated in both genre-load and bulk pack-load paths
