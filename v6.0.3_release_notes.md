# v6.0.3 Release Notes — Updater installs buzzer LaunchAgent

## Summary

`scripts/update-dockers.sh` now installs the buzzer macOS LaunchAgent as part of the chart-toppers update step. One command on a new (or existing) show machine pulls the repo, rebuilds the Docker stack, and (re)installs the buzzer service so it points at the new code path.

## Changes

- After `docker compose up -d --build` for chart-toppers, the script:
  1. Builds the buzzer Python venv if it's missing (`buzzer/venv/`)
  2. Runs `buzzer/install_service.sh` to (re)load `~/Library/LaunchAgents/com.twistedmelon.qlab-buzzer.plist` against the current repo path
- Skipped automatically on non-macOS hosts.
- WIIAM and Docforge are unaffected — they don't ship a buzzer.

## Usage on a new machine

```bash
mkdir -p ~/TwistedMelon
git clone https://github.com/TwistedMelonIO/chart-toppers.git ~/TwistedMelon/chart-toppers
~/TwistedMelon/chart-toppers/scripts/update-dockers.sh
```

That clones all three repos to `~/TwistedMelon/`, rebuilds each Docker stack, and installs the buzzer LaunchAgent — no separate buzzer install step.
