# v5.7.1 Release Notes — Genre picker shrink-rate split

**Released: May 1, 2026**

## What's New

### Genre picker font sizing — split threshold from shrink rate
- **Summary**: The genre picker's auto-shrink now has two independent knobs per resolution: a **no-shrink ceiling** (max length that renders at full size) and a **shrink-rate basis** (how aggressively longer labels shrink past the ceiling).
- **Details**: Previously a single `genreShrinkThresholds` value did double duty — raising it to fit one label always made longer labels too big to fit. Added `genreShrinkRateBasis` so the two concerns can be tuned separately.
- **Impact**: At `5520x1080`, `70'S, 80'S & 90'S` (17 chars) now renders at the full 80pt while `MEDITERRANEAN CLASSICS` (22 chars) still shrinks to 47pt and fits its button. All other genre labels (≤9 chars, e.g. `K POP`, `LATIN`, `BOY BANDS`) unchanged at 80pt.

## Technical Changes

- Added `GENRE_SHRINK_RATE_BASIS_DEFAULTS` constant (per-resolution).
- Added `packSettings.genreShrinkRateBasis` (per-resolution map), persisted to `pack-settings.json` alongside the existing `genreShrinkThresholds`.
- `computeGenreFontSize()` formula updated: `len <= threshold` returns base; otherwise `round(base * rateBasis / len)`.
- `genreShrinkThresholds['5520x1080']` raised 13 → 17 (no-shrink ceiling).
- `genreShrinkRateBasis['5520x1080']` set to 13 (preserves the original shrink-rate for >17-char labels).
