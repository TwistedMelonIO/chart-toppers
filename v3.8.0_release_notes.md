# v3.8.0 Release Notes — Bundled Buzzer & Round 4 Track Control

**Released: April 3, 2026**

## What's New

### Bundled QLab Buzzer
- **Summary**: USB buzzer support now ships with Chart Toppers
- **Details**: The QLab Buzzer bridge is bundled in `buzzer/` — no separate install needed. Key 1 triggers `IBUZZ` (Icon Buzzer), Key 2 triggers `ABUZZ` (Anthem Buzzer). Works globally even when QLab isn't the foreground app.
- **Impact**: Techs get buzzer support out of the box with the standard install

### One-Command Start/Stop
- **Summary**: `start.sh` and `stop.sh` scripts for the full system
- **Details**: `./start.sh` launches Docker containers and the buzzer bridge together. `./stop.sh` cleanly shuts both down. First run auto-creates the Python virtual environment and installs dependencies.
- **Impact**: Techs run one command to start everything — no Docker or Python knowledge needed

### Automated Buzzer Install
- **Summary**: `install_license.sh` now sets up the buzzer automatically
- **Details**: During first install, the script creates the buzzer virtual environment, installs pynput, and prompts about macOS Accessibility permissions
- **Impact**: Complete setup in one pass — nothing missed

### Round 4 Track Play Control
- **Summary**: OSC-triggered Round 4 tracks with replay prevention
- **Details**: Companion sends `/chart-toppers/r4/1` through `/r4/4` to play Round 4 tracks. Each track can only be triggered once — replays are rejected by the server. State resets when entering Round 4 or on game reset.
- **Impact**: Prevents accidental double-plays during the final round

## Technical Changes
- Added `buzzer/` directory with `qlab_buzzer.py`, `config.json`, `requirements.txt`
- Added `start.sh` — launches Docker + buzzer, creates venv on first run
- Added `stop.sh` — stops buzzer (via PID file) + Docker containers
- Updated `install_license.sh` with buzzer venv setup and accessibility prompt
- Added `r4TracksPlayed` state tracking with `playR4Track()` function
- Added OSC handler for `/chart-toppers/r4/[1-4]` addresses
- Updated `.gitignore` and `.dockerignore` for buzzer artifacts

## Setup Notes
- **Accessibility**: Terminal app must be added to System Settings > Privacy & Security > Accessibility
- **Python**: Requires Python 3.10+ (install via `brew install python@3.12` if needed)
- **Buzzer Config**: Edit `buzzer/config.json` to change key mappings or cue numbers
