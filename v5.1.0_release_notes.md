## What's New

### Round 1 Auto-Randomise
- **Summary**: R1 now auto-shuffles 6 random track pairs from the full R1 pool on round entry.
- **Details**: New `resetR1Tracks` function mirrors the R4 auto-shuffle pattern â€” no genre pick required, tracks are drawn from all R1 tracks regardless of genre. Runs automatically via `setRound(1)`.
- **Impact**: R1 is now a self-contained buzzer round â€” no operator intervention needed to load tracks.

### LEADERSD Fallback on Tied Scores
- **Summary**: LEADERSD no longer leaves a stale target when scores are tied.
- **Details**: `retargetLeaderSD` now falls back to `turnState.currentTeam` when `aScore === iScore`, so the cue always points somewhere sensible. Previously it bailed out and left LEADERSD at whatever it was pointing at from a previous round/session.
- **Impact**: In R2 entering with a 0-0 tie, LEADERSD correctly points to the coin-toss winner's SD page. QLab's Auto StreamDeck (Re-Target) cue now lands on the right team page.

### LEADERSD Retarget on Coin Toss
- **Summary**: The server now updates LEADERSD immediately when a coin toss resolves.
- **Details**: Added `retargetLeaderSD` calls to `/api/cointoss/:team`, `/chart-toppers/cointoss/{team}` OSC, and the auto-cointoss path inside `/chart-toppers/playing/{team}`. Covers both explicit and implicit coin-toss flows.
- **Impact**: R2 answer-page navigation via the LEADERSD-driven Auto StreamDeck cue is now reliable from the first team onward.

### R2 Refresh Tracks â€” State-Based Flow
- **Summary**: `/chart-toppers/refreshtracks` is now state-based and idempotent.
- **Details**: Replaced the brittle counter mechanism with logic driven by `r2PlayedTeams` membership (0 â†’ ignore, 1 â†’ swap tracks for opponent, 2 â†’ retarget R2GO2 â†’ AWO). On swap, the opponent is marked as played to cover QLab setups where `/chart-toppers/playing/{secondteam}` never fires.
- **Impact**: Repeated `/refreshtracks` calls no longer prematurely advance R2 to AWO. The handler also now updates `turnState.currentTeam`, navigates the SD, force-retargets LEADERSD, and switches the dashboard glow in one pass.

### R1/R2/R3 StreamDeck Flow Simplified
- **Summary**: Server-driven team-page navigation removed from R2 and R3.
- **Details**: The `/playing/{team}` handler no longer calls `navigateStreamDeck(TEAM_PAGES[teamId])` in R2 or R3. After the genre picker, the SD stays on SDE1; the QLab Auto StreamDeck (Re-Target) cue uses LEADERSD's target to drive team-page navigation only when needed.
- **Impact**: Cleaner separation of concerns â€” the server retargets cues, QLab fires them. No more unexpected SD page jumps during play.

### R2 Defensive Buzzer Disarm
- **Summary**: `/correct/{team}` and `/incorrect` now disarm IBUZZ/ABUZZ after every answer in R2.
- **Details**: R2 has no buzzer round, but stray Companion/QLab buttons may arm the buzzers. Defensive disarm catches this on every correct/incorrect while `currentRound === 2`. R3's per-track arm/disarm logic is untouched.
- **Impact**: Buzzers are guaranteed to stay silent throughout R2 regardless of external button config.

### R3 Auto-Advance Safety Net
- **Summary**: If `/chart-toppers/r3play/X` fires while the server isn't on R3, it auto-advances via `setRound(3)`.
- **Details**: Covers the case where QLab moves to R3 without firing `/chart-toppers/round/3` explicitly. Logs a warning so the desync is visible.
- **Impact**: R3 track plays, correct/incorrect handling, and genre loading all work even if the round-advance OSC is missing from the QLab/Companion setup.

## Bug Fixes

- **DUALGO armed outside R4**: `getDisarmedCues` polarity was inverted â€” DUALGO was left armed in R1/R2/R3 instead of being disarmed. - **Docker**: `docker-compose.yml` APP_VERSION bumped to 5.1.0, `QLAB_CUE_TEAM_SD` env var removed.
