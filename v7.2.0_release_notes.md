# v7.2.0 Release Notes — One-Shot Buzzer Host Setup

**Released: May 2, 2026**

## What's New

### setup-buzzer-host.sh
- **Summary**: A single command provisions any new show machine for the buzzer daemon.
- **Details**: `scripts/setup-buzzer-host.sh` installs Homebrew + Python 3.12 if missing, clones/pulls the repo, builds the venv, installs the LaunchAgent, copies the Python.app permission path to the clipboard, and opens Privacy → Input Monitoring.
- **Impact**: New machine setup is one curl command + two toggles. No more typing long paths or hunting through Settings.

## Technical Changes
- Added `scripts/setup-buzzer-host.sh` (idempotent).
- Switched preferred buzzer launch path from PyInstaller .app to direct venv Python via the existing `buzzer/com.twistedmelon.qlab-buzzer.plist`. macOS attaches Input Monitoring + Accessibility to the Homebrew Python framework binary, so permissions persist across all chart-toppers updates.
