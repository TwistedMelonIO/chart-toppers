# v7.3.4 Release Notes — /refreshtracks always reloads the same genre

**Released: May 2, 2026**

## What's Fixed
- **R2 refresh-tracks now always pulls fresh tracks from the same genre.** Previously the OSC was state-driven — if no team had played yet, it silently did nothing. Operator wants it to behave consistently every press: reload the current genre with fresh tracks and return the StreamDeck to SDE1, regardless of who has played.

## Technical Changes
- `src/server.js` `/chart-toppers/refreshtracks` OSC handler:
  - No longer calls the state-aware `handleRefreshTracks()`.
  - Always calls `loadGenreTracks(lastLoadedGenre.genreIndex)` and `navigateStreamDeck(1)`.
  - R2GO2 → AWO advancement is handled separately by `/chart-toppers/secondteamanswer`, so it's safe to drop the state machine here.
