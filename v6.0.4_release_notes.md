# v6.0.4 Release Notes — R4 zero-score pity time + README all-in-one updater

## Summary

Two changes:

1. **R4 zero-score pity time** — a team entering Round 4 with `earnedTime = 0` now gets a 10-second QLab countdown instead of a full 100-second video. Server-side score state is left at 0 (the dashboard still shows 0); only QLab's `loadActionAt` is overridden so the countdown video plays its last 10 seconds.
2. **README all-in-one updater section** — adds a paste-into-Terminal block walking the user through using `scripts/update-dockers.sh` to update Chart Toppers + WIIAM + Docforge in one command, including the buzzer LaunchAgent install.

## R4 zero-score detail

- Affects only `setRound(4)` (round entry / panic recovery re-send).
- Both teams' load cues are now sent unconditionally on R4 entry — zero-score teams get a 10s pity load, others get their `remainingTime`.
- Logged as `loadActionAt for {Team}: 10s pity (zero score)` in the chart-toppers container logs.

## Upgrade

```bash
docker compose up -d --build
```

License keys, machine IDs, pack settings, and activity log persist across the rebuild.
