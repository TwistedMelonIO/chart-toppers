Chart Toppers v2.2.0

Enhanced version with improved port management and conflict resolution.

## New Features
- **Port Conflict Resolution**: Updated port configuration to avoid conflicts with other applications
- **Flexible Deployment**: Changed from ports 3000/3001 to 3200/3201 for better compatibility
- **Improved Coexistence**: Can now run alongside other applications without port conflicts

## Port Configuration Changes
- **Main Web Dashboard**: Port 3000 → 3200
- **OSC Bridge Communication**: Port 3001 → 3201
- **Internal Bridge URL**: Updated to use new port scheme
- **OSC Input Port**: 53536 (unchanged for Bitfocus Companion compatibility)

## Core Features (from v2.1.0)
- **Professional Scoring System**: Multi-round gameshow scoring with team management
- **QLab Integration**: Complete OSC communication for cue control and feedback
- **Real-time Updates**: WebSocket-based live scoring across multiple devices
- **Premium UI Design**: Modern broadcast-quality scoreboard interface
- **Docker Deployment**: Containerized setup with persistent data storage
- **Multi-protocol Support**: OSC, WebSocket, HTTP API endpoints

## Technical Improvements
- **Enhanced Docker Configuration**: Optimized for production environments
- **Better Network Isolation**: Reduced port conflicts with other services
- **Maintained Compatibility**: All existing integrations remain functional
- **Improved Documentation**: Updated setup guides and configuration examples

## Deployment Benefits
- **Multi-application Support**: Can run alongside mugshot app and other services
- **Flexible Port Allocation**: Easy to customize for specific environments
- **Production Ready**: Optimized for professional gameshow environments
- **Backward Compatible**: Existing QLab and OSC integrations unaffected

## API Endpoints
- **Web Dashboard**: http://localhost:3200
- **OSC Input**: Port 53536 (UDP/TCP)
- **WebSocket**: Real-time updates on port 3200
- **REST API**: Complete scoring control and configuration

This release focuses on operational improvements and deployment flexibility,
making Chart Toppers more suitable for complex production environments where
multiple applications need to coexist without conflicts.
