# v6.1.2 Release Notes — Buzzer log redirect uses dup2

## Summary

Fixes the buzzer .app silently producing no log output. PyInstaller `--windowed` bundles detach stdout/stderr at the file-descriptor level, so reassigning `sys.stdout` in Python isn't enough. The redirect now uses `os.dup2` to redirect fds 1 and 2 directly to `/tmp/qlab-buzzer.log`.

## Upgrade

```bash
cd ~/chart-toppers && git pull && ./buzzer/install_service.sh
```

After install, the LaunchAgent restart should populate `/tmp/qlab-buzzer.log` with the bridge startup message instead of leaving it empty.
